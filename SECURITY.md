# Security Policy

## Reporting a Vulnerability

Please report suspected security vulnerabilities through GitHub's private vulnerability reporting for this repository.

- Do not open public issues for security reports.
- Include reproduction steps, affected components, impact, and any suggested mitigation if you have one.

## Scope

Security reports are most useful when they cover:

- authentication and authorization flaws
- sandbox or credential isolation issues
- remote code execution paths
- secret disclosure or privilege escalation
- package publishing or supply-chain risks

We will triage reports, communicate next steps through the advisory, and coordinate disclosure after a fix is available.
